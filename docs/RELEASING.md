# Releasing Cadenza

## Overview

Pushing a tag of the form `vX.Y.Z` triggers the release workflow
(`.github/workflows/release.yml`), which builds signed installers for all
three platforms and publishes them as a GitHub Release draft.

```
git tag v0.1.4 && git push origin v0.1.4
```

The draft must be reviewed and published manually on GitHub after the
workflow completes successfully.

---

## One-time setup: ed25519 signing key

Run this once on a trusted machine to generate the key pair:

```bash
cargo tauri signer generate -w ~/.cadenza-updater.key
```

The command prints the public key to stdout and saves the private key to
`~/.cadenza-updater.key`. Both are needed:

**1. Embed the public key in the repo** — open
`src-tauri/tauri.conf.json` and replace the empty `pubkey` field with
the printed value:

```json
"plugins": {
  "updater": {
    "pubkey": "PASTE_PUBLIC_KEY_HERE",
    ...
  }
}
```

Commit this change. The pubkey is not secret — it ships inside the
installer so clients can verify update signatures.

**2. Store the private key as GitHub Secrets** — in the repository
settings under *Settings → Secrets and variables → Actions*, create:

| Secret name        | Value                                              |
|--------------------|----------------------------------------------------|
| `TAURI_PRIVATE_KEY`    | Contents of `~/.cadenza-updater.key`           |
| `TAURI_KEY_PASSWORD`   | Password entered during `signer generate`      |

Keep `~/.cadenza-updater.key` offline or in a password manager. Losing
it means users can no longer receive signed updates for existing installs.

## Workflow jobs

| Job            | Runner           | Bundle    | Signing |
|----------------|------------------|-----------|---------|
| `build-windows`| `windows-latest` | NSIS      | ed25519 |
| `build-linux`  | `ubuntu-22.04`   | AppImage  | ed25519 |
| `build-macos`  | `macos-latest`   | DMG       | ed25519 |

Each job:
1. Builds `cadenza-cli` via `beforeBuildCommand` (lands in `target/release/`)
2. Runs `cargo tauri build` through `tauri-apps/tauri-action@v0`
3. Signs each bundle with the ed25519 key
4. Uploads installers + `.sig` files to the GitHub Release draft
5. Generates / updates `latest.json` (updater manifest) in the release

The three jobs share the same release via the tag name. `tauri-action`
merges per-platform update entries into a single `latest.json`.

---

## Updater manifest (`latest.json`)

`latest.json` is automatically generated by `tauri-action` with
`includeUpdaterJson: true`. It is uploaded to every release and contains
per-platform download URLs and ed25519 signatures. The app fetches this
file from:

```
https://github.com/williammb/cadenza/releases/latest/download/latest.json
```

If you fork the repository, update the `endpoints` array in
`src-tauri/tauri.conf.json` to point to your fork's releases.

## Bumping the version

Version is set in `Cargo.toml` (workspace) and auto-read by Tauri:

```toml
# Cargo.toml
[workspace.package]
version = "0.1.4"
```

After bumping, commit and tag:

```bash
git add Cargo.toml Cargo.lock
git commit -m "chore: bump version to 0.1.4"
git tag v0.1.4
git push origin main --tags
```

## Local signer verification

To verify a bundle manually after download:

```bash
cargo tauri signer verify \
  --pub-key "$(cargo tauri signer generate --print-public-key ~/.cadenza-updater.key)" \
  Cadenza_0.1.4_x64-setup.exe
```
